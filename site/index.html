how it works

a proxy that grows

your agent's requests pass through GreenCube on the way to the LLM. each request teaches it something new.

agent sends a request

your agent calls the OpenAI API like normal. it doesn't know GreenCube exists.

GreenCube intercepts

the proxy catches the request before it leaves your machine.

injects relevant memory

past mistakes, learned facts, agent preferences — anything relevant gets added to context.

forwards to the LLM

the enriched request goes to whatever provider your agent uses. OpenAI, Ollama, anything.

extracts new knowledge

the response comes back. GreenCube pulls out facts, patterns, and outcomes worth remembering.

agent gets the response

your agent gets the same response it would have gotten — but next time, it'll be smarter.

what actually happens

it gets smarter every day

this is what running GreenCube actually looks like. no promises — just what we've seen.

Day 1

it just remembers things

your agent runs, GreenCube stores what happened. next session, the agent has context it didn't have before. fewer repeated questions. fewer wrong assumptions.

Day 3

it stops repeating mistakes

you flagged a bad response on day 1. the agent hasn't made that mistake since. the memory system connected the feedback to the pattern and blocked it.

Day 7

it starts having preferences

the agent picks tools it's succeeded with before. it avoids approaches that failed last week. its competence scores shift based on real outcomes, not vibes.

Day 10

it spawns a specialist

the agent hits a task outside its competence. it creates a child agent with a narrow focus, delegates the work, and absorbs the result.

under the hood

9 systems. one creature.

each system runs independently. together they make something that feels less like a tool and more like a thing that's alive.

Memory

stores facts, preferences, and outcomes across sessions. keyword retrieval, injected into context automatically.

Self-verify

after completing a task, it checks its own work. catches obvious mistakes before you see them.

Competence

tracks what it's good at and what it's not. scores shift based on real task outcomes.

Drives

internal motivations that influence what the agent prioritizes. curiosity, helpfulness, caution — tunable.

Idle thought

when nothing's happening, it reflects on past interactions. sometimes it figures things out on its own.

Spawning

creates child agents for tasks outside its competence. depth-limited. capped at 10 globally.

Mood

an internal state that shifts based on success and failure. affects risk tolerance and verbosity.

Journal

writes daily summaries of what happened. you can read them. so can the agent, next session.

Mistake prevention

feedback you give gets stored as patterns. the agent checks new responses against them before replying.
