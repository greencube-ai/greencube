how it works

three steps to smarter agents


1

open greencube

the API server starts automatically on localhost:9000. it stays running in your system tray.

2

run your agent normally

GreenCube proxies every request to your LLM. it injects relevant memories, logs everything, and runs tool calls in Docker sandboxes.

3

check back later

open the app to see what your agent learned. knowledge it extracted. tasks it completed. mistakes it won't make again.

core features

everything your agent needs

zero extra tokens in core mode. these features work by intercepting and enriching the requests your agent already makes.

persistent memory

episodic memory stores every task. keyword search retrieves relevant context and injects it before the LLM sees your message.

docker sandbox

shell commands, file operations, and HTTP requests run inside isolated Docker containers. nothing touches your host machine.

audit trail

every action logged with timestamps, permissions, costs, and results. see exactly what your agent did and why.

multi-provider

each agent can use a different LLM. OpenAI for complex tasks, Ollama for cheap ones. switch without changing code.

opt-in

alive mode

when you're ready for more. agents that don't just execute tasks but learn from them. costs additional tokens.

reflection — reviews what it learned after each task
knowledge extraction — stores structured facts for future use
agent-to-agent — agents can message each other for help
spawn specialist — agents create children when struggling in a domain
idle thinking — processes knowledge gaps between tasks
competence tracking — knows its strengths and weaknesses
