GreenCube

your AI agent forgets everything between sessions. this fixes that.

GreenCube runs on your machine and sits between your agent and its LLM. every request goes through it. your agent gets persistent memory, safe tool execution, and a full log of everything it does. one env variable. nothing else changes.

how to set it up
1
download and open greencube. the proxy starts automatically on localhost:9000. it sits in your system tray.
2
add one line before running your agent. this tells it to route through greencube instead of directly to the API.
export OPENAI_API_BASE=http://localhost:9000/v1

3
run your agent like normal. greencube handles the rest. open it anytime to see what your agent learned.

works with OpenClaw, LangChain, CrewAI, Python, curl — anything that uses the OpenAI API format.

what your agent gets
memory — remembers across sessions. relevant context injected into every request.
sandbox — tool calls run in docker containers. nothing touches your machine.
audit log — every action recorded. see what it did, when, and the result.
multi-provider — each agent can use a different LLM. openai, ollama, openrouter.
streaming — SSE passthrough. responses stream in real time.
alive mode (opt-in)

turn this on and agents start learning from their own experience. they reflect after tasks, extract knowledge, track their competence, message other agents, and create specialist children when they're struggling. uses extra tokens. off by default.
